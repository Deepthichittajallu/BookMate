System Design - Interview Prep

1. Scalability

Horizontal vs vertical scaling. Use load balancers, replication, sharding.

2. Load Balancing

Distributes traffic evenly across servers. Examples: Round Robin, NGINX, AWS ELB.

3. Caching

Stores frequently accessed data. Use Redis or Memcached to reduce DB hits.

4. API Design

RESTful principles, stateless design, versioning, and rate limiting.

Common Interview Qs

  • Design Instagram login system
  • How to handle millions of requests?
  • When to use cache vs DB?
